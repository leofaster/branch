Esta primera tarea consiste en familiarizarse un poco con herramientas que deben ser comunes para el desarrollo de cualquier proyecto. En las siguientes instrucciones posiblemente hayan algunos términos desconocidos si nunca has trabajado con Git; queda de tu parte investigar lo que significan para poder realizar la actividad.

Como parte práctica de esta tarea, tendrás que seguir los siguientes pasos: 1. Crear una cuenta en github. 2. Realizar un fork en github de este repo: http://github/devlabs-usb/tarea-1. 3. Clonar tu propio fork usando git clone. 4. Hacer merge del branch conflict-branch que existirá en tu repositorio a tu branch gh-pages (ver git help checkout y git help merge). 5. Resolver los "merge conflicts" que surgirán al hacer el merge. En tu propio repositorio, rellena tu archivo "index.html" con las respuestas a las preguntas teóricas que se encuentran acá. 6. Hacer push de los cambios a "index.html" a tu repositorio cuando termines de rellenar las respuestas. 7. Enviarnos un correo a devlabs-usb@talpor.com con el URL de tu repositorio en GitHub, tu nombre y carnet. Bonus: Si entiendes como funcionan los git pages, mándanos el link a tu página de respuestas.
